**Phase 0 closed end-to-end on 2026-05-20.** Cold-start setup → wizard → `MainWindow`; subsequent cold start → DPAPI cache bypass OR login window → `MainWindow`; 15-min idle → auto-lock → login. ~104 automated tests, CI green.

**Phase 1 opened with Sprint 7** — format-agnostic parsing foundations (interfaces, Polish helpers, registry, dedup hash) plus a deterministic PKO BP **PDF** parser for the "Wyciąg z rachunku" layout. Manual verification then revealed the freely-available PKO export is **"Historia rachunku"** (CSV/PDF/XML), not the paid "Wyciąg z rachunku" — so the PDF parser is speculative (synthetic-verified only). **Sprint 8 closed that pivot** — `IStatementParser` / `IBankDetector` now operate on a format-neutral `StatementInput` (PdfPig dropped from `Coffer.Core`), the speculative PKO PDF parser is gone, and `PkoHistoriaCsvParser` parses the "Historia rachunku" CSV (Windows-1250, signed amounts, joined descriptions), verified by a committed synthetic golden CSV in CI and a gitignored real CSV manually (39 transactions). The AI fallback, Anonymizer CLI, and remaining PKO layouts follow. See [sprint-7/log.md](sprint-7/log.md) for the finding and the CSV schema.

**Phase 4 closed with Sprint 10** — hybrid auto-categorisation (deterministic rules + learned cache, falling back to batched AI behind a budget gate). **Phase 6 shipped with Sprint 11** — a post-login Dashboard summarising the current month and recent spending, charted with LiveCharts2 over server-side aggregations. With Phases 0, 1, 2, 4, and 6 done the desktop MVP is end-to-end: setup/login → import → categorise → review → dashboard.

**Phase 7 closed with Sprint 12** — a Polish-language chat assistant that answers questions about the owner's finances by calling four read-only EF-backed tools (`GetTotalSpent` / `GetTransactions` / `GetSpendingByCategory` / `GetMonthlyTrend`) through a tool-call loop, never inventing numbers, metering every turn into the cost ledger behind the monthly budget gate, with anonymised tool output and a collapsible per-message tool-trace. Two PRs (12-A plumbing+tools+`ChatService` → 12-B "Asystent" chat page + shell wiring). Anomaly/goals/receipt tools slot in at their own phases without touching the orchestrator.

**Phase 8 closed with Sprint 13** — anomaly detection and alerts. Five in-memory statistical detectors (high-amount z-score, new-merchant, category-spike, duplicate-payment, missing-recurrence) run over a rolling 30-day recent / prior-6-month baseline window; a signature-deduped detection service persists findings as `Alert` rows with templated Polish text (a dismissed signature is never resurrected), surfaced on an Avalonia **Alerty** page with accept/dismiss. An `AnomalyCommentator` then rewrites the title/description of the top 10 findings per run with a single budget-gated, anonymised, reasoning-tier LLM call metered once as `anomaly-comment`, falling back to the templated text on any failure. The `FindAnomalies` chat tool deferred in Sprint 12 lands here, returning active alerts whose period overlaps a date range. Two PRs (13-A engine+UI → 13-B AI commentary + chat tool). With Phases 0, 1, 2, 4, 6, 7, and 8 done the desktop app spans setup/login → import → categorise → review → dashboard → chat → alerts.

**Phase 9 closed with Sprint 14** — the financial advisor. Domain `Goal` / `GoalContribution` / `GoalSnapshot` feed a deterministic `GoalFeasibilityEngine` with one strategy per goal type (purchase, large-expense, emergency-fund, mortgage-prepayment, investment, long-term) plus a `MortgagePrepaymentCalculator`, all scored against a `FinancialContext` derived from the transaction history (6-month moving averages). An Avalonia **Doradca** page renders the projection with a live simulator slider, alternative scenarios, and a 12-month chart. On top, an `AdvisorReportGenerator` turns the engine's verdicts into LLM-written per-goal risks and grounded cutting suggestions (budget-gated, anonymised, metered once as `advisor-report`, engine-only fallback on any failure), persisted as a daily `AdvisorReport` by a once-a-day desktop-startup snapshot job; the `GetGoals` chat tool deferred in Sprint 12 lands here. **The engine calculates; the AI only explains** — every number is deterministic C#. Three PRs (14-A engine → 14-B Doradca page → 14-C AI + chat tool). With Phases 0, 1, 2, 4, 6, 7, 8, and 9 done the desktop app spans setup/login → import → categorise → review → dashboard → chat → alerts → advisor.

**Sprint 15 closed the bilingual UI (PL + EN) i18n** — a cross-cutting effort (not a roadmap phase) giving the whole desktop app a runtime language switch with no restart. An `ILocalizer` over `Strings.resx` (English neutral) + `Strings.pl.resx` (Polish satellite) is resolved in markup through an `{l:Localize}` extension that refreshes live on change; the choice persists in a plaintext app-data file readable before the DEK exists, so setup/login honour it too. Migration ran in three PRs: 15-A foundation + pilot (`MainWindow` nav + Settings switch), 15-B every remaining view + VM-formatted status/error strings + app dialogs, 15-C the deterministic engine-origin display text (goal scenario/risk labels emitted as stable codes and resolved at the VM boundary, keeping `Coffer.Core` presentation-free) plus a resx key-parity test and the `docs/conventions.md` policy update. Money/number formatting stays `pl-PL` + " zł" regardless of UI language (v1). **Deferred:** runtime localization of anomaly alert text (baked into the DB at detection time, AI-rewritten as single-language prose) is tracked as a follow-up (#128).
