- Problem: first manual run crashed with `InvalidOperationException: DEK is not available` when DI resolved `SetupWizardViewModel`. Root cause: EF Core's `AddDbContextFactory<T>` resolves `DbContextOptions<T>` eagerly when the factory is **requested** (not when it's used), and that path runs the registered `dekProvider`. Asking DI for the factory before `IDekHolder.Set` triggered `Get()`'s throw. → Resolution: changed `SetupService` constructor to take `Func<IDbContextFactory<CofferDbContext>>` instead of the factory directly, and registered the delegate in `AddCofferSetup` so the actual `GetRequiredService<IDbContextFactory<CofferDbContext>>` call happens **after** `IDekHolder.Set` inside `CompleteSetupAsync`.
