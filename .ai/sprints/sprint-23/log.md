A shared `BackupSnapshotWriter` owns the copy so the daily service and the pre-migration hook share one tested path; `PreMigrationBackup` is refactored onto it with behaviour unchanged (its byte-for-byte tests guard that). Daily snapshot writes `.tmp` then atomic `File.Move`; naming `coffer-{YYYY-MM-DD}.db`; retention by a **pure** `BackupRetention` (30d daily, 90d pre-migration — the pre-migration folder gains the rotation it lacks today), parsed from filenames so it needs no DB and works before login.
