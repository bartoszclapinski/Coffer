- Deferred follow-ups from PR #41 review (filed as chore issues against Sprint 7):
  - **Review point 1 — #42** — `AutoLockMonitor` stale-tick race on `Start`/`Stop` reuse. `Timer.Dispose()` does not wait for in-flight callbacks; with the singleton lifetime an old tick can land on the next session. Recommendation: per-`Start` runId token.
  - **Review point 2 — #43** — fire-and-forget `LogoutAsync` in `HandleLogout`. Two scenarios: immediate re-login races a partially-invalidated cache; Wyloguj-then-exit may **never** complete the cache invalidation. Recommendation: synchronous `.GetAwaiter().GetResult()` matching the existing `ResolveStartupWindow` pattern (cache I/O is sub-millisecond on Windows).
  - **Review point 3 — #44** — `HandleLogout` not guarded against concurrent invocation. Wyloguj + auto-lock landing in the same dispatcher tick can produce two `LoginWindow` instances. Recommendation: `_logoutInFlight` UI-thread bool (no lock).

Without it, both `LoginService` and `SetupService` happy paths (the most security-critical code in Phase 0) lack CI coverage. Scope: `IVaultPaths` interface in Core, `CofferVaultPaths : IVaultPaths` wrapping the existing static, refactor LoginService / SetupService / WindowsDpapiKeyVault / App routing, add integration tests against temp dirs.
